# 402. Remove K digits (medium)

# Ideas

This problem allows us to delete k numbers from a string of numbers so that the remaining number is the smallest. In other words, we have to keep the relative position of the original number unchanged.

Take the num = 1432219, k = 3 in the title as an example. We need to return a string of length 4. The question is: How can we find out what the four positions are in order?

(figure 1)

For the brute force method, we need to enumerate C_n^(n-k) kinds of sequences (where n is the length of the number), and compare the maximum one by one. This time complexity is exponential and must be optimized.

One idea is:

-Traverse from left to right -For each traversed element, we decide whether to discard or retain

The key to the question is: How do we know whether an element should be kept or discarded?

There is a pre-knowledge here: For the two numbers 123a456 and 123b456, if a> b, then the number 123a456 is greater than the number 123b456, otherwise the number 123a456 is less than or equal to the number 123b456. In other words, the size relationship between two same digits depends on the size of the first different number.

So our thinking is:

-Traverse from left to right -For the elements traversed, we choose to keep. -But we can selectively discard the previous adjacent elements. -The basis for discarding or not is as described in the above pre-knowledge.

Taking the example of num = 1432219, k = 3 in the title, the graphical process is as follows:

(figure 2)

Since there is no adjacent element to the left, there is no way to discard it.

(image 3)

Because 4 is larger than the adjacent 1 on the left. If you choose to discard the 1 on the left, it will make the remaining number larger (the number at the beginning has changed from 1 to 4). Therefore, we still choose not discard.

(Figure 4)

Because 3 is smaller than the adjacent 4 on the left. If you choose to discard the 4 on the left, it will make the remaining numbers smaller (the number at the beginning has changed from 4 to 3). So we choose discard.

. . .

The thinking behind is similar, so I won't continue to analyze it.

However, it should be noted that if the given number is a monotonically increasing number, then our algorithm will always choose not to discard. As required in this topic, we must always make sure to discard k contradictions.

A simple idea is:

-Once discarded each time, k minus one. When k is reduced to 0, we can terminate the traversal early. -And when the traversal is complete, if k is still greater than 0. Let's suppose that there are finally x remaining to be discarded, then we need to choose to delete the last x elements.

The above ideas are feasible, but slightly more complicated.

(Figure 5)

We need to reverse our thinking. Just now, my focus has always been discard, and the topic requires us to discard k. On the other hand, doesn't it mean that we keep n−kn-kn−k elements? Where n is the length of the number. Then we only need to intercept the first n-k elements after the traversal is completed according to the above method.

According to the above ideas, we choose the data structure. Since we need to reserve and **discard adjacent elements, it is more appropriate to use a stack, which is a data structure that adds and deletes at one end. Let's look at the code implementation.

# Code (Python)

class Solution(object):
    def removeKdigits(self, num, k):
        stack = []
        remain = len(num)-k
        for digit in num:
            while k and stack and stack[-1]> digit:
                stack.pop()
                k -= 1
            stack.append(digit)
        return''.join(stack[:remain]).lstrip('0') or '0'

Tip: If the title is changed to ask for the maximum number after deleting k characters, we only need to change the greater than sign in stack[-1]> digit to a less than sign.

# Ideas

Unlike the above question, this question does not have a global number of deletions k. Instead, there is a value of k for every letter c that appears in the string s. This k is the number of occurrences of c-1.

Following the above knowledge, the first thing we need to do is to calculate the k of each character. A dictionary can be used to describe this relationship, where key is the character c and value is the number of occurrences.

Specific algorithm:

-Build a dictionary. The key is the character c, and the value is the remaining number of occurrences. -Traverse the string from left to right, each time a character is traversed, the remaining number of occurrences-1. -For each character, if its corresponding remaining number of occurrences is greater than 1, we can choose to discard it (or not to discard it), otherwise it cannot be discarded. -The criteria for discarding are similar to the above questions. If the adjacent elements in the stack are lexicographically larger, then we choose to discard the elements in the adjacent stack.

Remember the boundary conditions of the above topic? If the remaining elements in the stack are greater than n−kn-kn−k, we choose to intercept the first n−kn-kn−k numbers. However, the k in this question is scattered among the characters, so this idea is not feasible.

But don’t worry. Because the title is required to appear only once. We can simply determine whether it is on the stack when traversing.

Code:

class Solution:
    def removeDuplicateLetters(self, s) -> int:
        stack = []
        remain_counter = collections.Counter(s)

        for c in s:
            if c not in stack:
                while stack and c <stack[-1] and remain_counter[stack[-1]]> 0:
                    stack.pop()
                stack.append(c)
            remain_counter[c] -= 1
        return''.join(stack)

A method to query whether a given character exists in a sequence. Basically, there are two possibilities:

-Ordered sequence: Dichotomy is possible, and the time complexity is roughly O(N)O(N)O(N). -Unordered sequence: You can use the traversal method. In the worst case, the time complexity is O(N)O(N)O(N). We can also use space for time, using the space of NNN in exchange for the time complexity of O(1)O(1)O(1).

Since the stack in this question is not ordered, our optimization point considers space for time. Since each character can only appear once, hashset can be used here.

# Code (Python)

class Solution:
    def removeDuplicateLetters(self, s) -> int:
        stack = []
        seen = set()
        remain_counter = collections.Counter(s)

        for c in s:
            if c not in seen:
                while stack and c <stack[-1] and remain_counter[stack[-1]]> 0:
                    seen.discard(stack.pop())
                seen.add(c)
                stack.append(c)
            remain_counter[c] -= 1
        return''.join(stack)

# Ideas

Similar to the first question, except that this time there are two arrays instead of one, and it is the maximum number.

The maximum and minimum are irrelevant. The key is that there are two arrays, and the number of elements selected from the two arrays is required to add up to k.

However, take k numbers in an array and keep it the smallest (or largest), we already know. But if the problem is extended to two, what will change?

In fact, the nature of the problem has not changed. Suppose we take k1 from nums1 and k2 from num2, where k1 + k2 = k. We will solve the two sub-problems of k1 and k2. Since these two sub-problems are independent of each other, we only need to solve them separately and then combine the results.

Suppose the numbers k1 and k2 have been taken out. Then all that remains to be done is to merge the numbers with lengths k1 and k2 into an array of length k into the largest array.

Take the title nums1 = [3, 4, 6, 5] nums2 = [9, 1, 2, 5, 8, 3] k = 5 as an example. If we take out 1 number from num1, then we have to take out 4 numbers from nums2.

Using the method of the first question, we calculate that [6] should be taken as nums1, and [9,5,8,3] should be taken as nums2. How to combine [6] and [9,5,8,3] to make the number as large as possible and keep the relative position unchanged?

In fact, this process is a bit similar to the ** rule ** in Merge Sort, and the above process of calculating the maximum number of num1 and num2 is similar to Score in Merge Sort.

We call the array consisting of k1 numbers selected from num1 as A, and the array consisting of k2 numbers selected from num2 as B,

def merge(A, B):
    ans = []
    while A or B:
        bigger = A if A> B else B
        ans.append(bigger[0])
        bigger.pop(0)
    return ans

# Code (Python)

class Solution:
    def maxNumber(self, nums1, nums2, k):

        def pick_max(nums, k):
            stack = []
            drop = len(nums)-k
            for num in nums:
                while drop and stack and stack[-1] <num:
                    stack.pop()
                    drop -= 1
                stack.append(num)
            return stack[:k]

        def merge(A, B):
            ans = []
            while A or B:
                bigger = A if A> B else B
                ans.append(bigger[0])
                bigger.pop(0)
            return ans

        return max(merge(pick_max(nums1, i), pick_max(nums2, ki)) for i in range(k+1) if i <= len(nums1) and ki <= len(nums2))
