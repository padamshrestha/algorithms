# Team game

The first programming problem is a team game problem.

# Title description

There are three teams, each of which is numbered team 1, team 2, and team 3. These three teams need to play n games in total. Now that k games have been played, each game cannot be tied, one point is awarded for winning a game, and no points are lost for losing. It is known that the score difference between team 1 and team 2 is d1 points, and the score difference between team 2 and team 3 is d2 points. Two teams can be selected for each game. Seeking if the last (nk) game is played, is it possible that the scores of the three teams will tie.

# Ideas

Suppose that the number of victories of team 1, team 2, and team 3 at this time are a, b, c, respectively, and the total number of victories of team 1, team 2, and team 3 are n1, n2, n3, respectively.

My initial idea was to ensure that n1, n2, and n3 are equal and less than or equal to n / 3. If the question gives the value of n1, n2, n3 directly:

print(n1 == n2 == n3 == n / 3)
1

But not only did n1, n2, and n3 not give, but also did not give a, b, and c.

In fact, our information at this time is only:

① a + b + c = k
② a-b = d1 or b-a = d1
③ b-c = d2 or c-b = d2
1
2
3

Where k and d1, d2 are known. a, b, c are unknown. That is to say, we need to enumerate all the possibilities of a, b, c, solve the equation to find the legal a, b, c, and the legal a, b, c are all less than or equal to n / 3.

Solving mathematical equations of a, b, and c is the difficulty of middle school mathematics. You can simplify the three equations. See the code area below for details.

-a only needs to win n / 3 again-a times -b only needs to win n / 3 again-b times -c only needs to win n / 3 again-c times

n1 = a + n / 3-a = n / 3
n2 = b + (n / 3-b) = n / 3
n3 = c + (n / 3-c) = n / 3

The problem is solved if it is abstracted. We only need to record whether the added window is 0 or 1:

-If it is 1, we don't have to do anything -If it is 0, we subtract 1 from m

Correspondingly, we need to record whether the removed window is 0 or 1:

-If it is 1, we do nothing -If it is 0, it means that when it is added, it is 1. When it is added, we subtract 1 from m. At this time, we add 1.

lucifer Tips: In fact, the question is to find the length of continuous a or b. Seeing continuity, everyone should also have the sensitivity of sliding windows, don’t care if it works, think there should always be.

Let's take A = [1, 1, 0, 1, 0, 1], m = 1 for example. Look at the specific process of the algorithm:

lucifer Tips: The number on the left represents the size of the window at this time, the yellow square represents the repaired wall, and the black square represents the window.

Here I visually regard 0 as a hole and 1 as a wall. Our goal is to fill the hole so that the continuous wall is the longest.

Every time we encounter a hole, we go to repair it indiscriminately. Since m is equal to 1, that means we can fill up one hole at most. Therefore, when repairing more than one hole, we need to adjust the window range so that at most one wall can be repaired in the window. Since the window represents a continuous wall (existing or repaired), we can finally return to the maximum value of the window.

Since there are two holes in the window below, this conflicts with "fill at most one hole". We need to shrink the window to meet the prerequisite of "fill at most one hole".

Therefore, the largest window is max(2, 3, 4, ...) = 4.

lucifer tip: It can be seen that we patched all the holes indiscriminately and adjusted the window so that there are at most m patched holes in the window, so the maximum value of the window is the answer. However, in fact, we don't need to really "fix" (0 becomes 1), but just modify the value of m.

class Solution:
    def longestOnes(self, A: List[int], m: int) -> int:
        i = 0
        for j in range(len(A)):
            m -= 1-A[j]
            if m <0:
                m += 1-A[i]
                i += 1
        return j-i + 1
